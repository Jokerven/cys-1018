Introduction

This website was created by putting considerable thoughts into the following steps: UI Design, Implementation, mobile responsiveness, and review.

UI Design

UI design, the process through which designers create user interfaces in software or digital devices with an emphasis on aesthetics or style. These designs provide a visual representation of user-friendly graphical interfaces. The UI design for this website was built on Whimsical after doing extensive research on different website thereby drawing design inspiration from a few websites, most of which came from adeolaadeoti.xyz

Implementation

The HyperText Markup Language (HTML) and Cascading Style Sheets (CSS) are two of the most important and fundamental web technologies. HTML is a set of markup codes that are placed into a file for internet display. The markup instructs the web browser on how to display the words and graphics on a web page. Using components, tags, and attributes, we are able to construct structural sections, paragraphs, and links. It's worth mentioning, though, that HTML isn't a programming language because it can't be used to create dynamic functionality.

CSS, on the other hand, is a basic design language that is used to control the appearance and feel of a webpage. You may modify the color of the text, the font style, the space between paragraphs and other style based modifications using CSS.

Mobile Responsiveness

A responsive web design is one that will automatically adjust for different screen sizes and viewports.

In this website, I added mobile responsiveness using css mobile query.

Drawbacks and Reflection

I did have some concerns about how to approach mobile responsiveness, starting with the mobile navigation without the usage of javascript to turn on and off the navigation menu. This was a little difficult, but after some research, I discovered that there was a method to achieve it using an html input checkbox for the toggle action and CSS to design the label that is bound to the checkbox as a hamburger menu on which the toggle is done. I found out that to achieve mobile responsiveness, I had to use "mobile first" design approach which simply entails that you have to create the web pages for mobile screens first before moving up to larger screen sizes

The project appeared challenging at first, but it was ultimately gratifying since it introduced me to the principles of the web and web technology in general. I had to learn how to use git commands and github for version control. I designed and constructed a website that was responsive, functional, user-friendly, and most importantly, useable. Over the course of this project, I've gained a wider understanding of web development, web technologies, and the fundamentals of software engineering.
